# 项目名称：流金梦坊 (OC Workshop)

**文档版本：** V1.0 (MVP 核心功能版)
**适用端：** 响应式 Web 应用 (PC + Mobile 浏览器适配)

## 1\. 产品概述

本产品旨在为网文作者、语C玩家及二次元爱好者提供一个集\*\*“角色深度创作 + AI辅助生成 + 互动社交”\*\*为一体的Web平台。用户不仅可以精细化打磨OC（Original Character）设定，还能利用Qwen3大模型快速完善人设、生成独立或多人互动的短篇故事，并以卡牌形式进行分享和跨用户联动。

-----

## 2\. 用户角色与核心流程

  * **创作者 (User)：** 创建OC、使用AI扩写、生成故事、导出文档、发布到广场。
  * **浏览者 (Visitor)：** 浏览广场公开OC、点赞评论、申请联动权限。

-----

## 3\. 功能需求详解

### 3.1 OC 创作与管理模块 (Core)

[cite_start]此模块需支持**半开放结构** [cite: 1]，满足从简略到复杂的各种设定需求。

  * **P0 - 基础信息录入：**
      * **固定字段：** 姓名、性别、年龄、种族、职业。
      * **图文混排区：** 外貌描写、性格特征、背景故事（支持富文本编辑器）。
  * **P1 - 自定义属性 (半开放特性)：**
      * **标签系统：** 用户可自由添加 Key-Value 标签，例如 `MBTI: INFJ`、`DND阵营: 混乱中立`、`战斗力: 5000`。
      * **可视化展示：** 自定义标签在前端以“Tag胶囊”形式展示。
  * [cite_start]**P0 - 图片管理 [cite: 2]：**
      * **上传：** 支持本地上传图片（自动压缩 + CDN存储）。
      * **素材库：** 内置系统默认头像库（二次元风格通用头像），供无图用户选择。
  * [cite_start]**P0 - 文档/卡片导出 [cite: 1]：**
      * **角色卡片：** 生成一张包含头像、关键属性、简介的精美图片（类似《原神》或《明日方舟》角色卡），用于朋友圈分享。
      * **文档存档：** 一键生成排版好的 PDF 或 Word 文档，保留所有详细设定文字。

### 3.2 DeepSeek AI 辅助模块 (AI Integration)

利用 DeepSeek API 实现“灵感填充”和“故事演绎”。

  * [cite_start]**P0 - 混合模式生成 OC [cite: 3]：**
      * **交互逻辑：** 表单中设置“随机骰子”按钮。
      * **锁定机制：** 用户填写“姓名：林克，性别：男，性格：沉默寡言”后，点击生成，AI 将自动补全“背景故事”、“外貌描写”等空白字段，已填写字段保持不变。
  * [cite_start]**P1 - 故事生成引擎 [cite: 3]：**
      * **单人小剧场：** 选中一个OC，输入关键词（如“雨夜、失恋”），AI 生成 500-1000 字短文。
      * **多人互动模式：**
          * **槽位选择：** 用户选择 `主角A (我的OC)` + `配角B (我的OC 或 获权的他人OC)`。
          * **Prompt 构造：** 系统自动提取 A 和 B 的人设卡作为 Context，要求 AI 编写两人互动剧情。

### 3.3 社交与广场模块 (Community)

[cite_start]基于**私密优先**和**许可联动**的社交逻辑 [cite: 4]。

  * **P0 - 隐私状态管理：**
      * **默认为私密：** 仅自己可见。
      * **公开/发布：** 用户手动点击“发布到广场”，OC 进入公共瀑布流。
  * **P1 - 社交互动：**
      * **基础互动：** 点赞、评论。
      * **联动申请 (核心特色)：**
        1.  用户 A 浏览用户 B 的 OC。
        2.  点击“申请联动”按钮。
        3.  用户 B 收到通知，批准后，该 OC 出现在用户 A 的“可用角色库”中（标记为“借用”）。
        4.  用户 A 可使用该 OC 进行 AI 故事生成。

## 4\. 界面与交互体验 (UI/UX)

[cite_start]**整体风格：** 二次元/游戏风（高饱和度、卡片质感、动态效果、Bilibili风格） [cite: 5]。

### [cite_start]4.1 移动端全功能适配方案 [cite: 5]

鉴于需要在手机上处理几千字的文本，设计需避免“长表单地狱”。

  * **方案 A：折叠手风琴 (Accordion Layout)**
      * 将设定分为“基本资料”、“外貌”、“背景”、“自定义”四个大折叠块。用户一次只展开一个模块进行填写，减少视觉压迫感。
  * **方案 B：沉浸式编辑器 (Focus Mode)**
      * 当用户点击“背景故事”这种长文本框时，弹出一个全屏的纯文本编辑层，屏蔽掉其他干扰元素，提供舒适的打字环境。
  * **方案 C：底部浮动操作栏 (FAB)**
      * 在移动端底部常驻“保存”、“AI润色”、“预览”按钮，确保长页面滚动时也能随时操作。

## 6\. 技术架构建议 (Technical Stack)

为了实现**腾讯云服务集成**与**Qwen**的快速接入，推荐以下架构：

  * **前端框架：** Vue 3。
      * *理由：* 组件化开发适合处理复杂的表单和卡片 UI，且生态丰富。
  * **后端服务：** 腾讯云 CloudBase (云开发)。
      * *理由：* 免运维，直接提供云数据库 (NoSQL) 存储非结构化的 OC 数据（JSON），云存储 (COS) 存放图片和生成的文件，云函数处理 Qwen API 的请求转发（隐藏 API Key）。
  * **AI 接入：** Qwen3 API。
      * *Prompt 策略：* 使用 System Prompt 约束 AI 输出 JSON 格式以便前端解析（用于填表），使用 Markdown 格式输出故事（用于阅读）。
  * **图片生成/处理：**
      * 前端使用 `html2canvas` 库，将 DOM 节点直接渲染为图片实现“卡片导出”。
